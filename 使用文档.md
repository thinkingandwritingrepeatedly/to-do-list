# 我的想法箱 - 使用文档

## 快速开始

### 启动服务

项目需要 Node.js 环境（建议 v18+）。打开命令行或双击脚本启动：

```bash
# 方式一：双击 start.bat（推荐）
双击 start.bat 文件

# 方式二：命令行指定端口
start.bat 4567

# 方式三：npm
npm start
```

启动后终端显示：
```
  ✓ 我的想法箱 服务已启动
  ✓ 数据存储于: D:\重要数据\todos.json
  ✓ 访问地址: http://localhost:3456
```

### 访问页面

打开浏览器访问 `http://localhost:3456`（端口号以实际启动时显示的为准）。

**注意**：必须通过 `http://localhost:端口` 访问，直接双击 `index.html` 会因为无法访问 API 而无法加载数据。

### 停止服务

在启动服务的命令行窗口中按 **Ctrl+C**，服务会自动关闭。

## 功能介绍

### 1. 快速添加想法
- 在顶部输入框中输入想法标题，按 **Enter** 键或点击 **「添加」** 按钮即可快速新增
- 点击右侧的 **「▼」** 展开详细选项，可以：
  - 选择或输入新的**分类**（如：工作、学习、生活等）
  - 添加**备注**文字
- 添加后输入框自动清空并保持聚焦，方便连续记录

### 2. 管理想法
每个想法支持以下操作：

| 操作 | 说明 |
|------|------|
| 切换状态 | 点击状态图标（圆形）可在「待办 ⇄ 已完成」之间切换 |
| 完成 | 标记为已完成 |
| 暂缓 | 标记为暂时不想做（已搁置） |
| 恢复待办 | 将已完成或已搁置的想法重新设为待办 |
| 编辑 | 修改标题、分类、状态、备注 |
| 删除 | 删除想法（带二次确认） |

### 3. 查看与检索

**状态筛选**
通过 **「全部 / 待办 / 已完成 / 已搁置」** 按钮快速筛选不同状态的想法。

**分类筛选**
点击顶部的分类标签，可以只看某个分类下的所有想法。再次点击取消筛选。

**搜索**
在搜索框输入关键词，实时搜索标题和备注内容。

**排序**
支持按最新优先、最早优先、按标题排序。

### 4. 统计信息
页面顶部实时显示：
- 想法总数
- 各状态数量（待办 / 已完成 / 已搁置）
- 今日/本周/本月新增数量
- 各分类条目数

## 数据存储

### 存储位置

所有数据存储在 **`D:\重要数据\todos.json`** 文件中。

相比传统浏览器 Local Storage 方案的优势：

| 特性 | 旧方案 (localStorage) | 新方案 (本地 JSON 文件) |
|------|----------------------|------------------------|
| 持久性 | 清除浏览器缓存即丢失 | 清除浏览器缓存不影响 |
| 跨浏览器 | 不互通 | 共用同一文件（服务相同即可） |
| 可查看 | 需开发者工具 | 文本编辑器直接打开 |
| 备份 | 手动导出 | 直接复制文件即可 |

### 存储空间

页面顶部会显示存储位置、当前数据文件大小以及**最后更新时间**（精确到秒）。

## 导出数据

点击页面顶部的 **「导出数据」** 按钮，将所有想法导出为一个 JSON 文件下载到本地。

导出的 JSON 文件结构：
```json
{
  "version": 1,
  "exportedAt": "2026-05-29T...",
  "todos": [
    {
      "id": 1748500000000,
      "title": "示例想法",
      "notes": "备注内容",
      "category": "工作",
      "status": "pending",
      "createdAt": "2026-05-29T...",
      "updatedAt": "2026-05-29T..."
    }
  ]
}
```

## 导入数据

点击页面顶部的 **「导入数据」** 按钮，选择 JSON 文件即可将数据导入。

支持的导入格式：
1. **导出的备份文件**：`{"version": 1, "todos": [...]}`
2. **纯数据数组**：`[{"id": 1, "title": "...", ...}]`

导入流程：
1. 点击「导入数据」按钮
2. 在文件选择窗口中选择 `.json` 文件
3. 系统自动校验文件格式和数据结构
4. 弹出确认对话框，显示将导入的数据条数
5. 确认后当前数据被**全部替换**为导入的数据

## 常见问题

### Q: 为什么双击 index.html 无法使用？
A: 项目依赖本地 HTTP 服务提供数据读写能力。必须先启动 `start.bat` 或 `node server.js`，然后通过 `http://localhost:3456` 访问。

### Q: 如何更换端口？
A: 运行 `start.bat 端口号`，例如 `start.bat 8080`。如果默认端口 3456 被占用，启动脚本会自动清理旧进程。

### Q: 如何备份数据？
A: 可以直接复制 `D:\重要数据\todos.json` 文件，或使用页面顶部的「导出数据」功能下载备份。

### Q: 如何迁移旧版 localStorage 中的数据？
A: 在旧版页面中点击「导出数据」下载 JSON 文件，然后在新版中点击「导入数据」选择该文件即可。

### Q: 清除浏览器缓存会导致数据丢失吗？
A: 不会。数据存储在 `D:\重要数据\todos.json` 中，与浏览器缓存无关。清除缓存最多导致需要重新登录页面。

### Q: 启动时提示端口被占用？
A: server.js 启动时会自动查找并清理占用同一端口的旧进程。如果自动清理失败，可以手动在任务管理器中结束残留的 node.exe 进程，或更换端口启动。

## 注意事项

- 需要 Node.js 环境（v18+），启动时自动检测
- 服务只能在本机访问（localhost），不对外网开放
- 建议使用 Chrome、Edge、Firefox 等现代浏览器
