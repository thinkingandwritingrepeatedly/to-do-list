# 我的想法箱 📝

一个轻量、快速、本地化的个人待办事项管理工具。采用**纯前端单页应用 + 本地 Node.js 服务**架构，数据通过 HTTP API 持久化到本地文件系统，彻底解决浏览器 Local Storage 因缓存重置导致的数据丢失问题。

> 数据掌握在自己手中 —— 所有数据存储在本地 JSON 文件中，随时可用文本编辑器查看和编辑。

---

## 功能特色 ✨

- **快速记录** — 顶部输入框随时记录想法，按 Enter 快速添加
- **状态管理** — 待办 / 已完成 / 已搁置，三种状态自由切换
- **分类组织** — 内置分类（工作/学习/生活/灵感/其他）+ 自定义分类
- **全文搜索** — 实时搜索标题和备注内容
- **多维筛选** — 按状态筛选 + 按分类筛选 + 多种排序方式
- **数据导出/导入** — 一键导出备份 JSON 文件，支持导入恢复
- **实时统计** — 总数、各状态数量、今日/本周/本月新增一目了然
- **连接检测** — 服务断开时自动提示，防止数据丢失
- **文件更新时间** — 显示数据文件的最后修改时间，精确到秒

## 快速开始 🚀

### 环境要求

- **Node.js** v18+（建议使用 [nvm-windows](https://github.com/coreybutler/nvm-windows) 管理版本）

### 启动方式

```bash
# 方式一：双击 start.bat（推荐）
双击 start.bat 文件

# 方式二：指定端口启动
start.bat 4567

# 方式三：npm
npm start
```

启动后终端显示：

```
  ✓ 我的想法箱 服务已启动
  ✓ 数据存储于: D:\重要数据\todos.json
  ✓ 访问地址: http://localhost:3456
```

### 访问页面

打开浏览器访问 **http://localhost:3456**

> ⚠️ 必须通过 `http://localhost:端口` 访问，直接双击 `index.html` 无法加载数据。

### 停止服务

在启动服务的终端中按 **Ctrl+C**

## 技术栈 🛠️

| 层级 | 技术 |
|------|------|
| UI | HTML5 + CSS3（Flexbox 布局） |
| 样式 | 原生 CSS（CSS Variables），零外部依赖 |
| 逻辑 | Vanilla JavaScript (ES6+) |
| 通信 | Fetch API（HTTP） |
| 服务端 | Node.js 内置 `http` 模块，零外部依赖 |
| 存储 | 本地文件系统（JSON），路径 `D:\重要数据\todos.json` |

## 数据存储 💾

所有数据存储在 **`D:\重要数据\todos.json`** 中。

| 特性 | 说明 |
|------|------|
| 持久性 | 清除浏览器缓存不影响数据 |
| 可查看 | 文本编辑器直接打开 |
| 备份 | 直接复制文件或使用导出功能 |
| 格式 | 标准 JSON 数组 |

## API 接口

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/` | 返回 index.html 首页 |
| GET | `/api/data` | 读取 todos.json 返回全量数据 |
| PUT | `/api/data` | 覆盖写入 todos.json |
| GET | `/api/info` | 文件信息（大小、路径、最后修改时间） |
| GET | `/api/health` | 健康检查（服务状态 + 时间戳） |
