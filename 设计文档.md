# 我的想法箱 - 设计文档

## 一、项目概述

一个轻量级的个人待办事项管理工具，采用纯前端单页应用架构，数据全部存储在本地浏览器中。旨在提供「随时记录、快速管理、定期回顾」的流畅体验。

## 二、功能需求

### 2.1 核心功能
| 功能 | 说明 |
|------|------|
| 快速新增 | 随时记录想法/任务，支持标题 + 分类 + 备注 |
| 状态管理 | 待办 / 已完成 / 已搁置 三种状态自由切换 |
| 编辑修改 | 支持修改标题、分类、状态、备注 |
| 删除 | 删除不再需要的条目（带二次确认） |
| 搜索 | 全文搜索标题和备注内容 |
| 筛选 | 按状态筛选（全部/待办/已完成/已搁置） |
| 排序 | 按创建时间、标题、分类排序 |
| 分类汇总 | 自动统计各分类下的条目数量 |
| 全局统计 | 总条目数、各状态数量统计 |
| 存储提示 | 底部显示数据存储位置及占用空间 |
| 数据导出 | 导出全部数据为 JSON 文件下载到本地 |

### 2.2 数据模型

```typescript
interface Todo {
  id: number;           // 时间戳 ID
  title: string;        // 标题
  notes: string;        // 备注
  category: string;     // 分类名称
  status: 'pending' | 'completed' | 'postponed';
  createdAt: string;    // ISO 创建时间
  updatedAt: string;    // ISO 更新时间
}
```

### 2.3 默认分类
- 工作 (蓝色)
- 学习 (绿色)
- 生活 (橙色)
- 灵感 (紫色)
- 其他 (灰色)

用户可在新增/编辑时输入自定义分类，系统自动识别并添加。

## 三、技术架构

### 3.1 架构选择
- **方案**：纯前端单页应用（Single Page Application）
- **文件**：单 HTML 文件（嵌入式 CSS + JavaScript）
- **理由**：无需服务器、无需安装、双击即可使用

### 3.2 技术栈
| 层级 | 技术 | 说明 |
|------|------|------|
| UI | HTML5 + CSS3 | 语义化标签 + Flexbox/Grid 布局 |
| 样式 | 原生 CSS（CSS Variables） | 无外部依赖，加载零延迟 |
| 逻辑 | Vanilla JavaScript (ES6+) | 无需框架，轻量高效 |
| 存储 | Web Storage API (localStorage) | 数据持久化，无数量限制 |

### 3.3 数据持久化
- 使用 `localStorage` 存储所有数据
- Key: `my-thought-box-todos`
- 每次数据变更自动保存
- 页面刷新后自动恢复数据

### 3.4 代码结构

```
index.html
├── <head>
│   ├── Meta (charset, viewport)
│   └── <style>           // 所有样式
├── <body>
│   ├── <header>          // 标题 + 全局统计
│   ├── <main>
│   │   ├── .quick-add    // 快速新增区域
│   │   ├── .categories   // 分类汇总
│   │   ├── .toolbar      // 搜索 + 筛选 + 排序
│   │   └── .todo-list    // 待办事项列表
│   ├── .modal            // 编辑弹窗
│   └── .toast            // 通知提示
└── <script>              // 所有 JavaScript
    ├── Store             // 数据管理 (CRUD + 持久化)
    ├── Renderer          // UI 渲染
    └── App               // 事件绑定 + 应用入口
```

### 3.5 数据流

```
用户操作 → 事件处理 → Store 更新数据 → localStorage 保存
                    → Renderer 重新渲染 → DOM 更新
```

## 四、UI/UX 设计

### 4.1 设计原则
- **清新简洁**：浅色主题、充足留白、柔和色彩
- **反馈即时**：操作后即时视觉反馈
- **高效录入**：支持 Enter 快速添加，输入框持续聚焦
- **渐进展示**：核心功能优先展示，次要功能按需展开

### 4.2 色彩方案
| 用途 | 色值 |
|------|------|
| 背景色 | `#F8FAFC` |
| 卡片背景 | `#FFFFFF` |
| 主色调 | `#5B7FFF` |
| 主色浅色 | `#EDF1FF` |
| 成功（已完成） | `#22C55E` |
| 警告（已搁置） | `#F59E0B` |
| 危险（删除） | `#EF4444` |
| 正文 | `#1E293B` |
| 次要文字 | `#64748B` |
| 边框 | `#E2E8F0` |

### 4.3 响应式
- 桌面端：最大宽度 720px，居中显示
- 移动端：自适应全宽

## 五、安全与边界

### 5.1 数据安全
- 数据仅存储在用户本地浏览器
- 不涉及网络传输
- 用户可手动清除浏览器数据来重置

### 5.2 边界处理
- 空状态：无数据时展示引导提示
- 搜索无结果：显示友好空状态提示
- 编辑时标题为空：阻止保存并提示
- 删除操作：二次确认弹窗
- localStorage 不可用时：降级提示
